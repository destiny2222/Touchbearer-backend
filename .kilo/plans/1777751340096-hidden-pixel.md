# Plan: Add Flexible Exam Timing for External Exams

## Goal
Implement flexible exam timing for external exams (e.g., entrance exams) where students can start the exam anytime within a window from `exam_date_time` to an optional `exam_end_datetime`, but must complete within `duration_minutes` once started. Ensure backward compatibility and prevent manipulation by enforcing time limits and single submission.

## Assumptions
- External exams are for `NewStudent` role, Internal for `Student`.
- Students can only submit once per exam.
- Time enforcement is on server-side to prevent client manipulation.
- Add optional tracking of time spent for results.

## Proposed Changes

### 1. Database Schema Changes
- Add nullable `exam_end_datetime` column to `exams` table (DATETIME type).
- Add `started_at` column to `exam_results` table (DATETIME, nullable) to record when student first accesses the exam.
- Add `time_spent_minutes` column to `exam_results` table (INT, nullable) to track actual time spent.

### 2. API Changes
- **POST /api/exams/store**: Accept optional `exam_end_datetime` in request body. Validate that for external exams, if provided, it must be after `exam_date_time`.
- **GET /api/exams/student/current-exam**: 
  - For external exams with `exam_end_datetime`, check if current time is between `exam_date_time` and `exam_end_datetime`.
  - On first access (no existing result), set `started_at` in `exam_results` to current time.
  - Return remaining time based on `started_at + duration_minutes`.
- **POST /api/exams/answers**:
  - Check if submission is within `started_at + duration_minutes` and before `exam_end_datetime` (if set).
  - Calculate `time_spent_minutes` as `submitted_at - started_at`.
  - Reject if time exceeded.
- Update result fetching endpoints to include `started_at` and `time_spent_minutes`.

### 3. Logic Adjustments
- For external exams:
  - Availability window: `exam_date_time` to `exam_end_datetime` (if set, else as before).
  - Once started (`started_at` set), enforce `duration_minutes` from `started_at`.
  - Submission window: From `started_at` to min(`started_at + duration`, `exam_end_datetime`).
- For internal exams: No change, keep fixed start time enforcement.
- Prevent multiple starts: `started_at` set on first access, persists across sessions.

### 4. Security Considerations
- Server-side time validation prevents client-side timer manipulation.
- Single submission enforced by existing check for existing results.
- If student tries to access after `started_at + duration`, deny or show expired.
- Logging suspicious activities (e.g., multiple rapid accesses) could be added later.

### 5. Backward Compatibility
- `exam_end_datetime` is optional; if null, behavior unchanged.
- Existing exams work as before.
- New fields in results are nullable.

### 6. Implementation Steps
1. Run database migration to add new columns.
2. Update exam creation validation and insertion.
3. Modify `/student/current-exam` to set `started_at` and calculate remaining time.
4. Update `/answers` to validate time and record `time_spent_minutes`.
5. Update result queries to include new fields.
6. Test with internal and external exam scenarios.

### 7. Potential Issues
- Student logs out and back in: Timer continues from `started_at`.
- Network issues during exam: If they exceed time due to lag, may need grace period (e.g., +5 mins).
- Timezone handling: Ensure all datetimes are in consistent timezone (UTC recommended).
